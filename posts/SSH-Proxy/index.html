Simply setting up a Proxy Server with SSH

Created2021-08-29|Updated2021-10-21|NotesComputer Science
|Word count:262|Reading time:1min|Post View:
Did you know that SSH can make your host as a proxy server?

SSH

Connecte to the remote host with SSH:

# ssh <user-name>@<host-ip>
# example
ssh alpaca@192.168.0.2
# the default port of SSH is 22, or you can use the parameter "-p" to specify the ssh port of the remote host.

TCP Port / Unix Socket forward

Forward local TCP Port or Unix Socket with parameter â€œ-Lâ€.

-L [bind_address:]port:host:hostport
-L [bind_address:]port:remote_socket
-L local_socket:host:hostport
-L local_socket:remote_socket

Specifies that connections to the given TCP port or Unix socket on the local (client) host are to be for-warded to the given host and port, or Unix socket, on the remote side.

# example
ssh -L 8877:192.168.0.101:7766 192.168.0.102

In the example above, it will forward the connections that given in arg( 192.168.0.102 ) from local port 8877 to remote server port 7766 , and compelete the progress.

Make the host as a proxy server with SSH (Tunnel)

In SSH commands, we can use parameter â€œ -D â€ to make a tunnel between local host and remote server.

Specifies a local â€œdynamicâ€ application-level port forwarding.

For example:

# ssh <user-name>@<host-ip> [-D [bind_address:]port]
# example
ssh alpaca@123.123.123.123 -D 1234

We made a tunnel that connects host 123.123.123.123 , and port of tunnel is 1234 .

Now, we can take localhost:1234 as a proxy server. (That can forward for our connection to 123.123.123.123:21 .)
